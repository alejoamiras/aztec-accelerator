## Item 1 ‚Äî F-003 Windows tail (bb prove workspace ACL)

**Lives in the `core` crate, not `src-tauri`.** `packages/accelerator/core/src/bb.rs`:
- `prove_tmp_parent()` (`bb.rs:82-100`): Unix branch `DirBuilder::new().mode(0o700)` + `set_permissions`; **Windows branch is a bare `create_dir_all(&base).ok()?` (`bb.rs:97-98`) ‚Äî no ACL.** Comment assumes "%LOCALAPPDATA% is already per-user" (assumption, not enforced).
- `create_prove_tempdir()` (`bb.rs:107-124`): `tempfile::Builder` with `#[cfg(unix)] .permissions(0o700)` (`:110-114`); **no `#[cfg(windows)]` branch at all.**
- `write_witness()` (`bb.rs:129-140`): `OpenOptions::new().create_new(true)` with `#[cfg(unix)] .mode(0o600)` (`:133-137`); **no Windows branch.** Witness file = `ivc-inputs.msgpack`.
- Existing test `prove_workspace_and_witness_have_private_modes` (`bb.rs:250-272`), `#[cfg(unix)] #[test]`, reads `MetadataExt::mode()` ‚Üí **the template to mirror with a `#[cfg(windows)]` twin that reads the effective DACL.**

**No existing Windows ACL/SID/SECURITY_DESCRIPTOR code anywhere in the Rust crates.** Every `#[cfg(windows)]` hit is behavioral (URL scheme in `windows.rs:23-30`, Task Scheduler XML in `crash_recovery.rs`). So the ACL logic is net-new; only the `#[cfg(windows)]` gating + test-shape conventions are reusable.

**Deps**: `windows-sys 0.61.2` + `windows 0.61.3` are present only **transitively** (via tempfile/tokio/tauri) in `core/Cargo.lock`. To write ACL code, add `windows-sys` as a **direct** dep to `core/Cargo.toml` with `Win32_Security`, `Win32_Security_Authorization`, `Win32_Foundation` (+ `Win32_System_Memory` for `LocalFree`) features. **`windows-acl` crate is NOT in the repo** ‚Äî using it = brand-new external dep (avoid per supply-chain caution; promoting the already-resolved `windows-sys` edge is cleaner). Align to the resolved `0.61.x` to avoid Cargo.lock churn.

**CI (decisive)**: `.github/workflows/accelerator.yml` job `windows-build` (`runs-on: windows-latest`, `:413-434`) runs `cargo test` for src-tauri **AND** `cargo test --manifest-path ../core/Cargo.toml` (`:433-434`), gated on `desktop` path-filter (any `packages/accelerator/**` change). **‚áí a `#[cfg(windows)]` test in `core/src/bb.rs` runs in real CI, no workflow change needed.**

**Reuse/adapt**: extract a pure `apply_owner_only_windows_acl(path: &Path) -> io::Result<()>` (testable directly on a tempdir/tempfile, matching `config.rs`'s `save_to`/`load_from` testability split). Wire into all 3 sites. Windows has no O_CREAT-mode equivalent ‚Üí ACL applied as a follow-up `SetNamedSecurityInfoW` (a narrow TOCTOU window worth calling out).

**Scope fork (flag for approval)**: `config.rs:171-174` and `certs.rs:251-252` **also** no-op perms on Windows ‚Äî `certs.rs` holds the **CA private key**. Audit F-003 named only the prove workspace, but a shared helper makes covering config + certs nearly free, and leaving the CA key world-readable on Windows is a worse inconsistency. Decide intentionally.

## Item 2 ‚Äî C9 authorize-popup edge cases

**(A) MED click-steal** ‚Äî `src-tauri/src/windows.rs`: `show_auth_popup_window` (`:109-160`) ‚Üí `open_or_focus_window` (`:61-87`) builds every popup `always_on_top(true)` + `.center()` + `set_focus()` on create. Per-origin dedup exists (`core/src/server/auth.rs:64-73`, `is_first` piggyback), but **distinct origins each get their own centered always-on-top focused window** (up to `MAX_PENDING_ORIGINS=10`, `authorization.rs:165`). No cross-origin serialization ‚áí popup #2 steals focus + identical screen position from popup #1. **No click-delay guard** in `bridge.js wireButton` (`:67-87`) ‚Äî it only disables buttons AFTER a click starts. Full serialization/queue = big lift + tangles with the 10√ó60s timeout math (`authorization.rs:253-255`). Proportionate fix = a click-delay "ignore input for ~N ms after show/focus" guard in `bridge.js`. Testable via a new WebDriver two-origin case (`auth-flow.spec.ts`) + a Rust unit assertion; NOT testable at the Playwright mock layer (single page-per-test).

**(B) LOW extension-scheme host validation** ‚Äî `core/src/authorization.rs:52-58`: the `chrome-extension|moz-extension|safari-web-extension` arm rejects a port and `to_ascii_lowercase()`s the host, but **`to_ascii_lowercase()` folds only A-Z**, leaving non-ASCII/bidi/zero-width/combining chars in the canonical string. `url::Url` treats these as opaque-host schemes ‚áí no IDNA/punycode (unlike http(s) at `:44-51`, protected per `canonical_origin_idn_punycode_no_homograph_collision`, `:625-637`). Real extension IDs are always plain ASCII (chrome: 32√ó `a-p`; firefox/safari: UUID-like). Fix = charset allowlist (reject non-`is_ascii_graphic`/bidi). Extend `canon_chrome_extension_*` tests (`:528-552`), `co()` idiom. Server ingress `auth.rs:36-42` calls `CanonicalOrigin::parse` ‚áí fix auto-hardens that path.

**(C) LOW get_pending_auth authoritative display** ‚Äî `frontend-src/authorize.js:3-8`: origin rendered from **URL query param only** (`params.get("origin")`); no server re-check. `get_pending_auth` **does not exist** (absent from `commands.rs`, `main.rs:487-500` invoke_handler, `capabilities/authorize.json:8`). `respond_auth` (`commands.rs:149-183`) already treats the origin arg as diagnostics-only (`:177`), resolving purely by `request_id`. Needed: `AuthorizationManager::peek_origin(&self, request_id) -> Option<CanonicalOrigin>` (non-consuming lookup on `by_request` map, `authorization.rs:181`; mirrors the `get_mut` at `:247`); `get_pending_auth(window, auth, request_id) -> Result<Option<String>,String>` bound by the **same** `auth-{hash(request_id)}` label check `respond_auth` uses (`require_label`, `commands.rs:163-164`); register in `main.rs`; add `allow-get-pending-auth` to `capabilities/authorize.json`; `authorize.js` calls it and trusts its answer (disable Allow on None/mismatch). Closest reuse template = `get_verified_info` (`commands.rs:136-147`). Playwright mock (`e2e/authorize.spec.ts:25-28` "shows decoded origin from URL params" + `e2e/tauri-mock.js` defaults) needs a `get_pending_auth` handler and the query-param test retitled.

**Collision risks**: any windowing change must not break the same-origin piggyback (`is_first`) or the request_id-keyed 60s timeout (`windows.rs:140-159`); `get_pending_auth`'s label check must stay in lock-step with `respond_auth`'s (same `sanitize_window_label`) or it leaks/refuses wrongly; if popups ever serialize, revisit the 10√ó60s timeout DoS math.

## Item 3 ‚Äî C8 autostart enable()‚ÜíResult rollback

`src-tauri/src/commands.rs:56-87` `set_autostart` enable branch order: `current_exe()` ‚Üí `autostart_path_is_safe` preflight (on unsafe: `manager.disable()` + `disable_crash_recovery()` + `Err`, `:73-78`) ‚Üí `manager.enable()` (plugin writes systemd/.desktop/plist/Run-key, `?`-propagated) ‚Üí **`enable_crash_recovery()` (returns `()`, swallows errors, `:81`)** ‚Üí `Ok`.

`crash_recovery.rs`: `CrashRecovery::enable(&self)` (`:16`) returns `()`; all 3 `enable_impl` (macOS `:62-90`, Linux `:171-249`, Windows `:321-380`) only `tracing::warn!` on failure ‚Äî **the residual**. Linux's actual arming step `systemctl --user enable` failure (`:237-244`) is swallowed. `disable_impl` already does remove-before-reload + confirmed-disarm bool (GATE-3 fold, `:254-282`) ‚Äî **reuse as-is for rollback**.

**Fix** (already specified in `clusters/C8-plan.md:63-66,195-199` + `C8-audit-gate3-codex.md:16-23`): make `enable(&self) -> Result<(),String>` + `enable_crash_recovery() -> Result<...>`; in `commands.rs`, on `Err` from crash-recovery-enable, roll back with the **same two-liner already at `:73-74`** (`manager.disable()` + `disable_crash_recovery()`) then return the error. **Reuse `CrashRecoveryGuard` RAII (`updater.rs:432-463`)** ‚Äî unit-tested with a `Cell` counter, no real `AppHandle` (`updater.rs:469-524`) ‚Äî as the testable-rollback precedent; factor a rollback helper taking injected closures so ordering/completeness is unit-testable on Linux CI.

**Must NOT hard-fail** (prior GATE-3 ask): the other two `enable_crash_recovery()` callers ‚Äî `main.rs:512-518` (startup rearm) and `updater.rs:411-419`/`:379` (`CrashRecoveryGuard` rearm) ‚Äî must **log-and-continue**, never abort startup / never falsely mark rearmed. Error conventions: commands use `Result<(),String>`; internal fns use `Result<(),Box<dyn Error+Send+Sync>>` `.map_err`'d at the boundary. Tests colocated `#[cfg(test)]`; only pure/injectable logic (`systemd_exec_start`, `autostart_path_is_safe`, `task_xml`, `CrashRecoveryGuard`) is covered today ‚Äî `enable_impl`/`set_autostart` have none. Playwright `e2e/settings.spec.ts:137-172` tests only the mocked frontend toggle.

**Out of scope (don't conflate)**: the AppImage/canonicalized-path preflight-vs-plugin mismatch (`C8-audit-gate3-codex.md` finding #1) is a separate documented deferral; `enable_safari_support` (`commands.rs:247-282`) is a sibling unrolled-back multi-step enable ‚Äî flag only, don't scope-creep.
