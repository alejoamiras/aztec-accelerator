- `healthy_aztec_on_port()` (main.rs:238-247) + `/health` (`status:ok, api_version:1, version`) = the "successfully launched" signal.

- Smoke scripts synth UNSIGNED feeds (`updater-smoke.sh:117` + siblings) with deterministic URLs → must consume pre-signed feeds once clients enforce.

## Decision ledger (3 plans → consolidation)
1. **Envelope encoding — RESOLVED to base64-verbatim (Fable) over typed-canonical (Codex).** The feed carries `manifest` = base64 of the EXACT signed bytes + `manifest_sig`. Verifier base64-decodes → verifies those exact bytes → strict-typed-parses the decoded envelope for binding. Reason: `raw_json` is parsed-not-bytes (Codex's own finding); a base64 STRING survives parsing intact, so verbatim decode has ZERO canonicalization drift. Codex's typed-object needs a byte-identical Rust canonicalizer (extra constraint). We STILL adopt Codex's strict typing (`deny_unknown_fields`, mandatory `u64` size, semver-canonical `version`, `BTreeMap` platforms) applied to the DECODED envelope.
2. **Same updater key** (both) — threat is feed-writer, not key theft; a 2nd key adds rotation surface, no threshold.
3. **Floor NOT in config.json** (both) — config load is fail-open; floor is fail-closed. New `updater-state.json`.
4. **`semver` crate** (both) — not `AztecVersion`.
5. **`VerifiedUpdate` newtype** (Codex) — only the verifier constructs it; `perform_update` accepts ONLY `VerifiedUpdate`.
6. **Shared Rust `examples/update-manifest.rs`** (Codex) — the ONE tool that assembles+signs+verifies the feed in CI (NOT jq/PowerShell/JS). `examples/` not `src/bin/` (bundle-shape, Cargo.toml:8-16).
7. **pubkey via `build.rs`** (Codex) → `env!("AZTEC_UPDATER_PUBKEY")` + `rerun-if-changed=tauri.conf.json` (single-source, build-validated). (Fable's `include_str!` is an equivalent fallback.)
8. **Floor bump = 3 consecutive version-matched `/health` probes** post-launch (Codex, stricter than Fable's single check).
9. **PR-safe fixture-key Linux rollback E2E** (Codex) in `accelerator.yml` — runs every PR, not just release. PLUS Fable's release-time `splice` smoke leg.

## Security & residuals
Splice/rollback/replay/tamper all fail-closed (envelope + floor). Signed mandatory `size` makes SEC-03's pre-flight cap trustworthy vs feed-writers; **residual**: plugin buffers to EOF before verify (no aborting callback) → a network-position attacker can still stream huge bytes for the artifact URL — unchanged, keep #345 open; post-download `len==size` rejects before install (integrity). Intermediate-feed replay/freeze = TUF territory (pub_date enables a future max-age; out of scope). Fresh-install: floor absent ⇒ only `>current` + envelope protect (bootstrap after healthy). Local same-user attacker (delete floor / swap binary) = out of scope (≡ local code exec). Same-key domain-separation via signed `schema` string + typed structure. Key rotation = old-key-signed bridge release embedding the new pubkey.

## Consolidated-plan audit (Codex xhigh) — verdict: REJECT → FOLDED (GATE 1 close-out)
Deep double-audit on the consolidation caught 3 blockers + corrections; all folded into the design below. Full transcript: C4-audit-codex.md.

**B1 — capability bypass (CRITICAL).** `capabilities/default.json:6` grants `updater:default` → frontend can call raw `check`/`download`/`install`/`download_and_install`, bypassing `VerifiedUpdate` entirely. FOLD: remove `updater:default` (or explicitly deny all `updater:*` commands) from capabilities; add a test/assert that no updater plugin command is frontend-reachable. All updates must flow through our Rust `check_for_update`/`perform_update`. `PendingUpdate` becomes `Option<VerifiedUpdate>` (private constructor + private inner `Update`).
**B2 — cross-process monotonic floor.** Process mutex is insufficient (multiple instances race installs/commits — real given multi-agent hosts; main.rs:223 server + main.rs:263 poller are independent). FOLD: OS-wide file-lock (flock) "updater transaction" guarding check→install→commit; persist a monotonic pending/accepted version; re-check floor UNDER the lock immediately before `install` (not just at perform_update entry, which is stale across a long download). Floor tracker keys off ACTUAL HTTP bind success (not "spawned"). 3 health probes → specify spacing + minimum dwell + reset-on-failure.
**B3 — restore outer-feed binding.** Require: strictly-typed outer `version`, `pub_date`, and the ENTIRE `platforms` projection == the decoded signed envelope; EXACT canonical `Update.version == envelope.version` (not merely `semver`-equal); `manifest_sig` defined as Tauri `.sig` base64, decoded once + length-capped; `notes` stays unsigned. (Selected-artifact match alone permits unsigned platform permutation + pub_date divergence.)
**Corrections folded:** Rust example ASSEMBLES/VERIFIES, Tauri CLI SIGNS (plan text corrected); scope signing secrets to the single signer step; add `sign-update-feed` to release-smokes + `verify-live-feed.needs`; use `Version::cmp_precedence` + restore the build-metadata-only rejection test; E2E tamper = in-place byte-flip (preserve length + genuine sig); Windows synthetic N-1 = `createUpdaterArtifacts=false` + keep prod pubkey (do NOT patch N-1 to a throwaway pubkey); splice leg asserts `SECURITY:` log + zero artifact requests; SEC-03 residual = "compromised artifact origin/CDN/TLS terminator" (not ordinary network attacker), post-download size check does NOT mitigate buffering DoS; "all replay fails closed" → only replay ≤ current/floor fails (intermediate-feed replay is the documented TUF residual).
**Assumptions corrected:** "successful launch signal" is an INFERENCE; `config.rs` pattern needs strengthening (predictable temp path, ignored chmod failure, no fsync at config.rs:146) — the new `updater_state.rs` must add: random temp name, chmod-failure = hard error, fsync file+dir. Signing-secret availability is a BLOCKING release prerequisite. F-005 is independent and must NOT gate C4. Key rotation: restore a second-channel/dynamic-pubkey migration (old-key bridge alone strands clients who skip it) — documented as a follow-up, not claimed solved.
