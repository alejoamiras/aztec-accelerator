C9 — The "which site is asking?" popup (F-014)

When a website asks to use the accelerator, a popup shows the site's origin and Allow/Deny. If that origin is displayed in a way that hides the real identity, a user could Allow a malicious look-alike. F-014 fixes how the origin is shown + how "Remember" behaves.

What we're fixing

  • Show the WHOLE origin, never truncated. The original plan tried a middle-ellipsis, but the audit proved that can't reliably keep the important part (the registrable domain) visible — e.g. tenant.github.io, ports, IPv6, long names all break it, and you can't fit an unbounded host in a fixed width. The correct, simpler fix: show the complete canonical origin as real, selectable, scrollable text (with Allow/Deny always reachable in a footer). Nothing is hidden, so nothing can be disguised. No 200KB public-suffix-list needed.
  • Keep punycode as-is (don't decode it to Unicode homoglyphs) + isolate text direction so bidi tricks can't visually reorder it.
  • "Remember" defaults to OFF. A hasty Allow shouldn't permanently trust a site. The primary button becomes "Allow once"; a separate "Always allow this site" checkbox is the deliberate opt-in.
  • Show the TRUE origin, not a spoofable one. The popup will display the origin the server holds for this request (via a new get_pending_auth command), not a URL parameter that could disagree with the request actually being resolved.

Deferred (documented follow-ups)

The audit also found two bigger, real issues tracked as follow-ups: stacked popups can "steal" a click by grabbing focus (needs UI serialization), and extension-scheme origins aren't ASCII-validated. Both are beyond this light cluster.
