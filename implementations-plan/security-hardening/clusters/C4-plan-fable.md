# C4 fable plan — key decisions (full text in the deep-audit transcript)
Verified ground truth: tauri-plugin-updater 2.10.1 + minisign-verify 0.2.5; NO fork (verify-after-check in check_for_update + re-verify in perform_update; Update.raw_json carries the whole feed); feed jq-generated in release-accelerator.yml:641-709 inside the AWS/contents:write release job (signing must move out); smoke scripts synth unsigned feeds (must pre-sign once clients enforce); config.rs:142-177 atomic-0600 pattern; config load is FAIL-OPEN so the floor must NOT live in config.json; AztecVersion ordering is lexicographic (use the `semver` crate); healthy_aztec_on_port() = the "successful launch" signal; pubkey via include_str!(tauri.conf.json).

DECISIONS:
- Layer A: embed the signed envelope bytes VERBATIM base64 in the feed (`manifest` + `manifest_sig`) — verifier decodes+verifies the EXACT bytes, NO canonical re-serialization (kills the drift landmine). New updater_manifest.rs::verify_manifest(raw_json, pubkey, claimed_version, download_url, artifact_sig) -> {version,size}. Require semver(env.version)==semver(claimed); exactly one platform entry with url==download_url, signature==artifact_sig, size present. b64 length cap pre-decode. allow_legacy=false. Called in check_for_update AND perform_update. Replace advertised_update_size trust base with the SIGNED size + post-download bytes.len()==size check.
- Same updater key (threat is feed-writer, not key theft).
- Layer B: new version_floor.rs, ~/.aztec-accelerator/version_floor.json atomic-0600 (copy config.rs pattern). load -> Missing|Valid(semver)|Corrupt. Corrupt ⇒ check returns None + never overwrite. candidate > max(semver(current), floor) in check AND perform. running<floor ⇒ disable+SECURITY log (debug-downgraded). Bump only after 30s grace + healthy_aztec_on_port() (crashing build never advances). semver handles rc precedence.
- Phase 3: sign-manifests job (needs [validate,build], permissions contents:read ONLY, no AWS/id-token) → read .sig+sizes → jq envelope + latest.json + per-platform smoke feeds → `bunx tauri signer sign` → self-verify with minisign CLI → upload artifact. release job DELETES its jq block + consumes artifact. verify-live-feed asserts manifest+sig. Add a `splice` negative smoke leg (outer version 999 + intact N envelope ⇒ /health stays N + SECURITY reject) — day-one Layer A teeth. Tamper = in-place byte flip (equal length). NEVER prod-sign a fake version.
- Local (GUI-less): all Phase 1-2 pure Rust; interop via dev-dep `minisign = "0.7"` OR committed fixtures. CI-only: updater E2E (macOS/Win/Linux), lint:actions, real-key release smoke.
- Residuals: intermediate-feed replay/freeze = TUF territory (pub_date enables future max-age); SEC-03 streaming bytes unchanged (upstream). Fail-closed inventory documented.
- Asks: F-005 lands independently; fail-closed UX (tray text proposed); minisign dev-dep vs 7-day min-age; second distribution channel.
