First-run onboarding wizard — 4 layout variants

Static mockups for the Aztec Accelerator's first-launch setup screen. All four cover the same three consented choices — Encrypted Connection (HTTPS), Start on Login, Auto-Update — but differ in flow and altitude. The HTTPS toggle is the consequential one: turning it on installs a local certificate into your browsers' trust stores, which raises a native OS dialog (macOS keychain password / Windows security prompt; silent on Linux). Pick a direction, or mix. Copy shown is real, not lorem.

A · Single card. Exactly what you described: all three toggles on one screen, each with a one-line explanation, one Start CTA. Lowest friction, everything visible at once. Best when we trust users to read three lines. This is the recommended baseline.

Partial-failure behavior (applies to every variant)

If the certificate step fails or is declined, HTTPS is left off with a Retry, and the other choices (Start on Login, Auto-Update) are still applied. The wizard never rolls everything back because one step failed.
Naming decision this asks you to make (plan Ask A1)

The mockups use "Encrypted Connection (HTTPS)" as the primary label (config key https_enabled, commands enable_https/disable_https). Alternatives considered: "Enable HTTPS" (clearer for failure states, per codex, but jargon-y as a headline) and "Encrypted Communication" (rejected as vague). The recommendation is the compromise: friendly label, plain command names. Tell me if you'd rather lead with "Enable HTTPS".
