Remove SPONSORED_FPC_SALT + refresh docs

SPONSORED_FPC_SALT was a secret we used to keep one address private — the "sponsored fee payer" the playground uses so users don't pay gas. Now that the standard (salt=0) fee payer is deployed and funded publicly on the v5 testnet, hiding its address buys nothing. So we rip out the secret everywhere (it's pure dead weight now), tidy up the docs that went stale during the 5.0 work, redeploy the playground, and delete the secret.

Why this tier was chosen

Light — none of the six risk dials are high. It's documentation plus removing an obsolete config value; the address it "protected" is now intentionally public. Single low-risk pass: one plan, one codex audit, ship.

Phases

P1 — Remove the secret from the code

The playground (and an SDK test) read SPONSORED_FPC_SALT to pick the fee payer. Drop that — always use the standard one (salt=0). No behavior change: that's already the default when the value is empty.

Proves itself by: lint + typecheck + unit + playground build all green; no reader left.

P2 — Remove it from CI + scripts

Eight GitHub workflows pass the secret around; a script sets it. Strip all of that out (keeping the unrelated --salt command-line flag, which deploys a custom fee payer — a real dev tool).

Proves itself by: actionlint + lint green; zero references left in .github/ + packages/.

P3 — Refresh 5 docs

Fix what went stale in the 5.0 work: add the key "you don't re-release the app for an Aztec bump — it downloads the prover at runtime" note; correct a wrong headless-server caveat (it no longer needs WebKit/GTK); document the SDK-publish release flow; refresh the root + playground READMEs; fix stale SDK code examples.

Proves itself by: review — each doc matches reality, no broken links, no leaked local paths.

P4 — Land the PR

Branch → PR → CI green (now running fee-payer-less) → auto-merge.

Proves itself by: SDK/App/Accelerator/actionlint gates green; PR auto-merges.

P5 — Redeploy playground + smoke (you-ish)

Redeploy the live playground (no SDK re-publish) and confirm in a browser that a deploy still works paying via the standard fee payer.

Proves itself by: deploy green; live bundle has no baked secret; a manual browser deploy mines.

P6 — Delete the secret

Once nothing references it, delete the GitHub secret.

Proves itself by: it's gone from the secret list; next CI run still green.

Human context

A "sponsored FPC" pays transaction fees for users so the demo is free. Its address is fully determined by a public contract + a salt; salt=0 is the canonical one. We'd been treating that salt as a secret out of caution. But the canonical fee payer is now deployed and funded for everyone to use, so the secret protected nothing — and it was about to cause confusion (and a stale "Required" line in the docs). Removing it is simpler and more honest.
