## Central thesis (all 3 planners converged)
**The `Host`-header allowlist is the keystone.** DNS rebinding works only because the rebound page's `Host` is `evil.com:59833` while the server gates on `Origin` alone. An exact loopback-`Host` check at ingress defeats rebinding *regardless of Origin*, on every route, desktop + headless — and (opus's key insight) it is **behavior-preserving** (every real caller already sends a loopback Host; only forged/external Host is newly rejected). So it ships first and alone; the breaking changes layer on top of it.

### PR-1 ✓ — `Host` allowlist ingress guard (SEC-01a) — *behavior-preserving, highest-leverage*  [MERGED #338]

## Security & Adversarial Considerations
- **Threat model:** malicious web page, DNS-rebinding page, local-foothold process, network MITM, compromised upstream `bb` release. The localhost server is the primary attack surface; the auto-updater (pinned minisign) is the secondary.
- **After each fix, the attacker's next move:** post-PR-1 rebinding is dead → attacker needs a local foothold (SEC-04 residual, documented). Post-PR-2 `/health` reveals at most "accelerator exists" (intentional, the SDK needs zero-config detection). Post-PR-3 an IPC caller can't resolve another's request without the opaque id (residual: code already able to invoke trusted Tauri commands). Post-PR-4 updater attacker moves from memory-DoS to signed-artifact compromise (acceptable — minisign is the authenticity control). Post-PR-5 `bb` attacker still wins on upstream GitHub compromise (SEC-02 debt, explicit). 
- **Host-check correctness is load-bearing:** must reject `[::ffff:127.0.0.1]`, decimal/hex IPs, `0.0.0.0`, absent/empty Host, wrong-port. Do NOT rely on `User-Agent`/Fetch-Metadata (Safari coverage uneven).
- **Least privilege / crypto:** no new secrets; the updater minisign pubkey is build-time-embedded (already pinned); the self-managed download must re-use the SAME minisign verification, not a weaker check. Cert CA stays keyless + name-constrained (unchanged).
- **Supply chain:** `bun.lock` frozen; any new dep (`reqwest` is already a dep) respects the 7-day min-age; no new npm publish surface.
