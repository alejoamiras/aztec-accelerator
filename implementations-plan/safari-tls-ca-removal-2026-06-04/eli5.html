Removing the risky "master key" from Safari HTTPS

The accelerator runs a tiny web server on your own machine. Safari refuses plain http://localhost for some features, so the app serves https instead — and for Safari to trust that, the app currently makes its own little certificate authority (think: a master key that can vouch for certificates) and tells your Mac to trust it. The problem: that master key is saved as a plain file on disk. Anything running as you that reads it could forge a "trusted" certificate for any website — your bank, your email — not just localhost. This plan throws the master key away the instant it's been used once. We still make the master key, use it to stamp one "localhost" certificate, and then destroy the key immediately — it's never saved to disk. Your Mac still trusts the setup exactly like before (so Safari is happy), but there's no longer any key lying around for someone to forge with. (We first tried removing the master key concept entirely, but Apple's own guidance says Safari needs that "authority" structure to trust a local site — so we keep the structure and kill the key instead. Safer and simpler.)

Human context

A normal HTTPS certificate proves "I am localhost." A certificate authority (CA) is one level up: it's a key that signs other certificates, so trusting it means trusting everything it signs. The app used a CA because it let the app quietly refresh its localhost certificate every couple of years without bugging you. But a trusted CA key on disk is a loaded gun: whoever holds it can mint trusted certificates for anything. We don't need that power — we only ever need one certificate, for localhost. So we drop the CA and trust the localhost certificate directly.

One Apple rule shapes the design: macOS won't accept a TLS certificate that's valid for more than ~2 years (825 days) — even one you trusted by hand. So the localhost certificate has to be replaced every couple of years, which means a quick "trust this?" prompt then. You decided that small occasional prompt is an acceptable price for removing the master key.

Phases

Phase 1 — Make the cert, then destroy the master key

Make the master key in memory, use it to stamp one "localhost" certificate (valid 824 days), write the certificate to disk — and never write the master key. It's discarded the moment it's done its one job. Share one helper for the cert settings so "make it" and "renew it" can't drift.

Phase 2 — Trust + renew safely

Trust the (now keyless) authority exactly like before — Safari already accepts this, no gamble. Renewal makes a brand-new authority + cert (since the old key is gone), so it needs a fresh "trust this?" prompt — we ask ~30 days before the old one expires (while it still works), add the new one before removing the old, and if you cancel, keep serving the old one. No silent renewals, no sudden "HTTPS broke" surprise, and old trust entries get cleaned up so they don't pile up.

Phase 3 — Clean up the old master key (upgrades)

For people upgrading: delete the old master-key file from disk. That alone defuses the danger — a trusted-but-keyless certificate can't forge anything. (We leave the old trust entry in your keychain so there's no scary prompt; there's an optional "remove it" button for a full cleanup.)

Phase 4 — Tidy up the file handling

Write cert files safely (all-or-nothing, owner-only permissions), and actually check a cert is valid (not just "a file exists") so a half-written file can't wedge HTTPS forever.

Phase 5 — Prove it (automatically)

Tests that generation writes the cert but never the master key, that renewal works and cleans up, and that upgrades delete the old key — no manual Safari testing needed.

Open questions

  • None blocking. The earlier worry — "will Safari trust this?" — is resolved: by keeping the authority structure (just without its key), we stay on Apple's recommended path, so trust works exactly as it does today.
  • Residual we're accepting: someone who copied the old master key before you upgraded keeps a copy until the old trust entry is removed (the optional cleanup button). And every ~2 years a renewal asks you to approve once.
