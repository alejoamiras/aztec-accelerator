**Plan**
The cleanest implementation is: make the invariants typed first, keep public surfaces stable, and use façade files/helpers so the “behavior-preserving” refactors stay mechanically checkable by the compiler and the existing E2E suite. The only intentional runtime change is F-02: every approved origin, including headless `ALLOWED_ORIGINS`, must enter the system as one canonical value.

- `F-02`: replace raw strings with a typed canonical origin everywhere. Add `#[derive(Debug, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)] #[serde(transparent, try_from = "String", into = "String")] pub struct CanonicalOrigin(String);` with `impl CanonicalOrigin { pub fn parse(input: &str) -> Option<Self>; pub fn as_str(&self) -> &str; }`, `impl TryFrom<String> for CanonicalOrigin`, `impl From<CanonicalOrigin> for String`, `impl AsRef<str>`, `impl Display`. Change `AcceleratorConfig.approved_origins` to `Vec<CanonicalOrigin>`, `AuthorizationManager.pending` to `HashMap<CanonicalOrigin, ...>`, and the auth signatures to take `&CanonicalOrigin`. `authorize_origin()` parses the `Origin` header into `CanonicalOrigin`; “remember” pushes typed values; `respond_auth()` and `remove_approved_origin()` re-parse the Tauri string arg before resolve/remove; `server/src/main.rs` parses `ALLOWED_ORIGINS` through `CanonicalOrigin`. I would also add `fn parse_allowed_origins_env(raw: &str) -> Result<Vec<CanonicalOrigin>, String>` as a pure seam. Existing coverage already hits canonicalization and auth flow; new tests should cover `CanonicalOrigin` serde round-trip on persisted canonical strings, canonicalization edge cases, and env parsing. Validity: compiler forces every comparison/storage site to use the newtype; existing auth tests in `authorization.rs` and `server.rs` stay green; WebDriver auth flow stays green. The risky part is serde migration: if you use bare `Vec<CanonicalOrigin>` deserialization, one stale non-canonical entry could default the whole config. Safer plan: keep the newtype, but deserialize the field via a compat helper that canonicalizes per-entry and preserves current drop/dedupe semantics while eliminating the post-load migration pass.

**Security**
- IDN/punycode: `CanonicalOrigin` must normalize through `url::Url`; add a unit case for an IDN hostname and lock the stored form. If `Url` yields punycode, that defeats homograph duplicates cleanly.
- Uppercase host/scheme: lowercased before storage, so `HTTPS://EXAMPLE.COM` and `https://example.com` collapse.
- Trailing dot: strip one canonical trailing dot and reject empty-after-strip hosts, so `example.com.` cannot evade exact-match approval.
- Default ports: elide `:80`/`:443`; preserve non-default ports; this removes alias pairs like `https://host` vs `https://host:443`.
- Scheme smuggling: accept only the explicit origin schemes already allowed; reject prefix lookalikes and unsupported schemes.
- Userinfo/path/query/fragment: reject, don’t ignore, so `https://trusted.com@evil.com`, `https://host/path`, and query-bearing inputs never enter the approval set.
- `\0` and whitespace: env parsing should trim surrounding whitespace before canonicalization; embedded control chars should fail header parsing or URL parsing and be rejected. Add tests for this corpus.
- Headless bypass closure: canonicalizing `ALLOWED_ORIGINS` does not change the core trust model. Missing `Origin` still means “browser-origin gating does not apply,” which is the documented localhost assumption; the fix only removes the raw-string bypass from the exact-match path.
