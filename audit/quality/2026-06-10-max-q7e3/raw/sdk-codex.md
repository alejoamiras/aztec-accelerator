**Findings**

1. Public shared types live in the hotspot class file. Smell: `Misplaced-Shared-Types` with a latent `Cyclic Dependency`. Impact: High | Blast radius: medium-wide | Frequency: high (`accelerator-prover.ts` is the 14-commit hotspot). Instances: `accelerator-prover.ts:11-92`, `accelerator-transport.ts:3,30-31,53,66,74,89`, `index.ts:1-9`. `AcceleratorTransport` depends on `AcceleratorProtocol`/`AcceleratorStatus`, but those types are owned by the prover implementation file, so internal transport edits and public-type edits now pull on the same unit and reinforce a 2-way edge. Smallest safe refactor: `Move shared types to types.ts` and re-export unchanged from `index.ts`. What disappears: the back-import, the latent cycle, and unnecessary churn in the prover file for transport-only work.

2. `#probeAndParseHealth` is a Long Method hiding a missing status factory. Smell: `Long Method` + `Duplicate Code` / missing `Factory`. Impact: High | Blast radius: wide | Frequency: high. Instances: `accelerator-prover.ts:233-328`; manual union construction at `248-253`, `269-274`, `291-298`, `308-314`, `317-323`, `326`. One method probes, parses JSON, distinguishes legacy vs multi-version protocol, applies version policy, logs, mutates protocol state, caches, and hand-builds six `AcceleratorStatus` variants. Adding one union field or status variant forces branch-by-branch edits and makes omissions likely. Smallest safe refactor: `Extract Method` for parse/policy steps plus `Introduce Factory` for status creation. What disappears: repeated object literals and branch-local knowledge of the public union shape.

3. Protocol pinning relies on branch-order side effects. Smell: `Temporal Coupling`. Impact: High | Blast radius: medium | Frequency: high. Instances: pin at `accelerator-prover.ts:256`; clear at `268` and `325`; non-pin error branch at `245-253`; later consumption in `accelerator-transport.ts:57-63,121-133`. The “don’t pin on error / clear on malformed JSON / clear on probe failure” rule is spread across exit paths, while `/prove` later reads that hidden state indirectly through `baseUrl`. Any new health branch must remember the pin/clear policy, or future maintenance silently bakes in stale negotiation state. Smallest safe refactor: `Move Method`/`Extract Method` so transport owns the protocol-state transition policy behind one API. What disappears: scattered `setProtocol(...)` calls and comment-driven invariants.

4. `createChonkProof` is a Divergent Change hotspot. Smell: `Divergent Change` (also a `Long Method`). Impact: Medium-high | Blast radius: wide | Frequency: high. Instances: `accelerator-prover.ts:330-398`, especially phase emission at `340,350,357,362-363,381,393,396`, fallback routing at `343-346,369-383`, and response/header handling at `387-398`. UI phase policy, availability fallback, 403 denial semantics, request serialization, prove-duration extraction, and proof decoding all change the same method for different reasons. Smallest safe refactor: `Extract Method` into remote prove, unavailable fallback, and duration/response handling helpers. What disappears: mixed concerns in the main orchestration path.
